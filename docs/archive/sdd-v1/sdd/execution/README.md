# Execution Layer

Use this folder after the root context and governance are loaded.

This is the canonical operating layer for task routing and execution contracts.

It does not replace:

- `AGENTS.md`
- `docs/sdd/context/`
- `docs/sdd/governance.md`
- `docs/sdd/governance/`
- `docs/specs/<feature-id>/`

## Canonical Files

- `task-routing.md`: choose the governed path, practical mode, primary contract, and primary output
- `task-input-header.md`: canonical request header, including the optional agent-profile selector
- `contracts/`: minimum execution contracts by task family

## Rule

Prompts are adapters. Execution contracts are operational truth.
