## Findings for SPOR00701AC

### 1. [P1] Create-new misbinds alarm columns

- File: `src/main/java/jp/co/brycen/kikancen/spor00701accreate/process/spor00701acCreateNewProcess.java`
- Relevant lines: 74-76, 122-124, 191-201
- Issue:
  - The insert column order is `AIRLN, ALARMKBN, ALARM`.
  - The binding block writes `dto.getAlarmInput()` into the first placeholder.
  - The actual `ALARM` column is then cleared with `ps.setEmpty(...)`.
- Risk:
  - New records persist alarm text into the wrong column.
  - The business warning text is lost from `ALARM`.
  - Any later logic that reads `ALARM` will not see the stored warning.

### 2. [P2] Dirty-check misses delivery-date-only edits

- Files:
  - `src/main/webapp/angular/src/app/components/spor00701ac/spor00701ac.component.ts`
  - `src/main/webapp/angular/src/app/components/spor00701ac/spor00701ac.component.html`
- Relevant lines:
  - template date input around line 315
  - `fnChangeDateInput()` at 1578-1593
  - `fnGetCurrentInputDto()` at 1869-1883
- Issue:
  - `fnChangeDateInput()` validates `funcDeliveryDateInput.getVal()` but does not sync `this.deliveryDateInput`.
  - `fnGetCurrentInputDto()` compares the cached `this.deliveryDateInput`.
  - Create and update requests use the live control value instead.
- Risk:
  - If the user changes only the delivery date, the unsaved-change check can incorrectly treat the screen as unchanged.

## Findings for SPOR01101AC

### 1. [P1] Detail quantity updates do not feed downstream processing

- File: `src/main/java/jp/co/brycen/kikancen/spor01101accreate/process/Spor01101acCreateUpdateProcess.java`
- Related files:
  - `src/main/java/jp/co/brycen/kikancen/spor01101acgetdetail/process/Spor01101acGetDetailProcess.java`
  - `src/main/java/jp/co/brycen/kikancen/spor01101acsearch/process/Spor01101acSearchAllRecProcess.java`
  - `src/main/java/jp/co/brycen/kikancen/spor01101acupdate/process/Spor01101acUpdateProcess.java`
- Relevant lines:
  - update writes `ORDERREQUESTQTYCHANGES` at `Spor01101acCreateUpdateProcess.java:41`
  - detail reads `ORDERREQUESTQTY` at `Spor01101acGetDetailProcess.java:231`
  - search projects `ORDERREQUESTQTY` at `Spor01101acSearchAllRecProcess.java:252`
  - bulk registration aggregates and inserts from `ORDERREQUESTQTY` at `Spor01101acUpdateProcess.java:481` and `Spor01101acUpdateProcess.java:357`
- Issue:
  - The detail update writes the edited quantity to `ORDERREQUESTQTYCHANGES`.
  - The reviewed detail/read/search/register paths continue reading `ORDERREQUESTQTY`.
- Risk:
  - An edited quantity can disappear when the row is reopened.
  - Bulk registration can generate orders from the pre-edit quantity.

### 2. [P1] Search SQL becomes malformed for valid partial date filters

- File: `src/main/java/jp/co/brycen/kikancen/spor01101acsearch/process/Spor01101acSearchAllRecProcess.java`
- Relevant lines: 210-221, 582-593
- Issue:
  - The SQL opens `AND ( (` only when `orderRequestDateFromSearch` is present.
  - It closes the full date group only when `deliveryDateToSearch` is present.
  - Parameter binding mirrors the same conditional structure.
- Risk:
  - Valid partial combinations such as only an order-request date range can produce unbalanced SQL and fail at runtime.

## Cross-function / module-level risks

### 1. Warning logic is duplicated and inconsistent in SPOR01101AC

- Files:
  - `src/main/java/jp/co/brycen/kikancen/spor01101accheckWarning/process/Spor01101accheckWarningProcess.java`
  - `src/main/java/jp/co/brycen/kikancen/spor01101acupdate/process/Spor01101acUpdateAlarmProcess.java`
- Observed inconsistencies:
  - One path uses `CURDATE()` where the other uses `SYSDATE()`.
  - Only one supplier-item calendar path applies `DELFLG`.
- Risk:
  - The same row can produce different warning results depending on whether it is checked in detail mode or bulk registration mode.

### 2. Backend validation depends heavily on frontend DTO shaping

- Files:
  - `src/main/java/jp/co/brycen/kikancen/spor00701accreate/process/spor00701acCreateProcess.java`
  - `src/main/java/jp/co/brycen/kikancen/spor01101accreate/process/Spor01101acCreateProcess.java`
- Issue:
  - Some create paths call `.toString()` on DTO values before defensive null handling.
- Risk:
  - Non-Angular callers or malformed requests are likely to fail with server errors instead of validation responses.

### 3. Shared optimistic-lock behavior amplifies field-mapping mistakes

- File: `src/main/java/jp/co/brycen/kikancen/common/process/MasterCheckExclusionProcess.java`
- Issue:
  - Both features rely on shared exclusion handling based on `UPDDATETIME` and `DELFLG`.
- Risk:
  - Persistence-field mismatches can surface later as exclusion conflicts rather than earlier contract failures.
