# Prompt: Create Or Update Feature Package

Use this prompt only after routing is already known.

## Canonical Backing Sources

- `docs/sdd/execution/contracts/create-spec.md`
- `docs/sdd/spec-authoring/README.md`
- `docs/sdd/execution-profiles/<current-agent>.md`

## Primary Output

- updated governed feature package under `docs/specs/<feature-id>/`

## Short Support Uses

- requirement clarification when the task is to turn raw notes into approved `REQ`, `DES`, `TASK`, `AC`, and `TC` structure
- document refactoring when an existing governed artifact needs clearer structure without changing approved meaning

## Keep The Variable Packet Small

- current ambiguity or raw input excerpt
- target artifacts to update
- explicit non-changes
- open questions that still block safe authoring

Do not author specs directly from raw notes without the spec-authoring workflow.
