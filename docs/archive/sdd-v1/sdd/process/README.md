# SDD Process

## When To Use

Use this folder to follow the repository's delivery sequence from intake through closeout.

## How To Use

1. Follow the numbered stages in order.
2. Use `05-self-review.md` and `99-ai-checklist.md` as the mandatory AI control loop before final output.
3. Use the built-in two-pass and done guidance inside the main stage files when the task needs extra control.
4. Do not skip gates just because the work is AI-assisted.

## Stage Order

1. `01-intake.md`
2. `02-requirements.md`
3. `03-design.md`
4. `04-task-planning.md`
5. `05-implementation.md`
6. `06-tests-and-verification.md`
7. `07-review.md`
8. `08-release.md`

## Mandatory Control Files

- `05-self-review.md`
- `99-ai-checklist.md`

## Gate

No stage is complete until its required output exists and the matching governance or checklist gate passes.
