# Acceptance Criteria

## AC-01

- Requirement links: REQ-01
- Statement: The repository contains a tracked documentation structure that clearly exposes SDD entrypoints, ADRs, specs, and changelog history.
- Evidence needed: presence and linkage of `README.md`, `docs/`, `docs/decisions/`, `docs/specs/`, and `CHANGELOG.md`

## AC-02

- Requirement links: REQ-02, REQ-03
- Statement: The SDD system defines a strict stage order from requirements through review, and provides templates and checklists for each stage.
- Evidence needed: process docs, templates, and checklists under `docs/sdd/`

## AC-03

- Requirement links: REQ-03
- Statement: Review and governance expectations are actionable at pull request time.
- Evidence needed: governance docs and PR template

## AC-04

- Requirement links: REQ-04
- Statement: The repository contains migration guidance, canonical standards, and at least one real example feature package using the new structure.
- Evidence needed: migration docs, standards, and this feature package

## Evidence Expectations

- Automated: not applicable
- Manual: direct inspection of created docs and links
- Review: PR template and governance alignment
