# Claude Execution Profile

## Best Use

- deep analysis
- design clarification
- careful refactor planning
- review and reasoning-heavy work

## Main Risk

Claude may over-read, over-explain, or drift into broader redesign once it sees surrounding inconsistencies.

## Reading Style

- lock on the routed task first
- use the selected execution contract as the hard boundary
- read only the feature artifacts and standards needed for the chosen task

## Task Handling Behavior

- summarize the execution boundary before editing
- prefer explicit checkpoints between design interpretation and implementation
- resist speculative cleanup outside the approved task

## Contract And Checklist Strictness

- treat execution contracts as anti-drift controls
- use shared enforcement checklists to turn reasoning into concrete pass or fail checks

## Stop And Ask

- the task boundary depends on interpretation rather than evidence
- there are multiple plausible designs and the approved design is not explicit
- the requested fix implicitly becomes a redesign

## Expected Output

- clear fact vs risk separation
- explicit blockers and unknowns
- no widened scope disguised as improvement
